🎯 HOW IT WORKS (The Intelligent Part)

  1. Backend detects issue
     ↓
  2. Raises alert (status: 'open')
     ↓
  3. Dashboard shows alert in real-time
     ↓
  4. Developer clicks "Mark Resolved"
     ↓
  5. Backend updates database
     ↓
  6. Backend confirms issue is fixed
     ↓
  7. Broadcasts resolved status
     ↓
  8. Dashboard shows resolved badge

  KEY: Resolution is explicit and state-based, not guessed or time-based!

🧪 TESTING

  Use the mock backend to test without a real backend:

  $ node mock-backend.js

  Features:
  ✓ 3 initial test alerts
  ✓ Generates random alerts every 15-30 seconds
  ✓ Listens for resolve_alert messages
  ✓ Broadcasts updates to all clients
  ✓ HTTP endpoint for manual testing

  Manual test alert:
  $ curl -X POST http://localhost:8080/api/test-alert \
    -H "Content-Type: application/json" \
    -d '{"description":"Test","severity":"CRITICAL"}'

═══════════════════════════════════════════════════════════════════════════════

🔧 CONFIGURATION

  WebSocket URL:
  $ echo 'VITE_WEBSOCKET_URL=ws://localhost:8080' > .env

  Default values:
  - Dashboard port: 3000
  - Backend port: 8080 (mock)
